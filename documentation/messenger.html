Messenger

First, you need to create the characters that will appear on Messenger. Go to the "characters" folder. Once you've done that, go to "talks" to create your conversations.

Characters

Open the characters.txt file located in messenger/characters/characters.txt.

Girlfriend "Sarah" (gf.png) = gf
Benjamin (benjamin.png) = be
Ophelie (ophelie.png) = op
Lisa (lisa.png) = li
Stephane (stephane.png) = st
Robert (robert.png) = ro
Element Description
Girlfriend "Sarah" The first line is the MC's girlfriend. Just edit her name between the quotation marks, and that will be the default name.
(gf.png) Profile picture in parentheses (square image recommended, but the code automatically adapts)
= gf Abbreviations after the "=" are useful for coding dialogues more quickly
Avatar Images

The character images must be placed in the "avatar" folder where characters.txt is located.

Conversations

Your first conversation should be in start.txt in the talks/ folder.

mc : $pics = 1.png

Line Meaning
Sarah The first line refers to the person with whom you are having the conversation. You can have a conversation with several people by separating their names with commas (e.g., Sarah, Robert, Lisa)
mc : text To make the MC speak, use "mc: TEXT" (respect the spaces)
gf : text To make a protagonist speak, use "Name: TEXT" (or "Abbreviation: TEXT")

Name Variables

You can use dynamic variables to display the MC's or girlfriend's custom name:

Variable Description
$mc Displays the MC's custom name
$gf Displays the girlfriend's custom name

Multi-line Messages

As long as another protagonist does not speak, all the text you enter, including line breaks, will be in a single message:

mc : Hey!
gf : Hey John, hwr?

Did you do what I asked you to do?

I haven't heard from you.
mc : I completely forgot.

This will only result in three messages.

Navigation Between Conversations

Once you have finished the conversation and want to start a new one, enter $talks = PATH/NAME.txt

From the "talks" folder, you can create subfolders. For example, if you create "lovepath/chapter1/1.txt", the call will be:

$talks = lovepath/chapter1/1.txt

If the MC already has a conversation with the person(s) in this new conversation, it will be reused and moved to the top of the MC's list, like a "new message" in WhatsApp.

Content Unlocking

You can unlock social media posts from within conversations:

Command Description
$insta = NAME.txt Sends a notification to the MC of the InstaPics post. They can click on it to be redirected to it. As long as a post is not called in a conversation, it will not appear in the application.
$slut = NAME.txt Same functionality as InstaPics but for OnlySlut

Media

You can send photos, videos, and voice notes in conversations.

Command Description Required Folder
$pics = NAME.png Send a photo (you can use different image formats) pics/ folder next to the .txt file
$vids = NAME.mp4 Send a video vids/ folder next to the .txt file
$audio = NAME.mp3 Send a voice note audio/ folder next to the .txt file

Example

gf : Look what I found today!
gf : $pics = surprise.jpg
mc : Wow, that's amazing!
gf : I even recorded a voice message for you
gf : $audio = message.mp3

Status

The $status command displays a centered bubble in the conversation, typically used for dates, times, or system messages.

Command Description
$status = TEXT Displays a centered status bubble with the specified text

Example

$status = Monday, 9:00 AM
mc : Good morning!
gf : Hey! Ready for the day?

$status = 2 hours later
gf : How's work going?
mc : Pretty busy, but I'm managing.
Tip

Use status messages to indicate time jumps, scene changes, or important moments in the story.

Typing Indicator

The $typing command displays a typing indicator (3 bouncing dots) before a message appears, simulating someone typing in real-time.

Command Description
$typing Typing indicator with random duration (1-2.5s)
$typing 2000 Typing indicator with specific duration (2000ms)
$typing = gf Typing indicator with speaker name (hover to see who's typing)
$typing 2000 = gf Full format: duration + speaker

Basic Example

mc : Hey, are you there?
$typing
gf : Yes! Just got home.
$typing 1500
gf : What's up?

Group Conversation Example

$typing = sarah
sarah : Did you see what happened?
$typing 2000 = emma
emma : No, what?
$typing = sarah
sarah : Check InstaPics...
Behavior

The typing indicator automatically advances to the next message after the duration expires. In group conversations, specifying the speaker lets users hover to see who is typing.

User Controls

Users can click to skip the typing animation and immediately see the message. In fast mode, typing animations are automatically skipped.

Message Deletion

The $delete command deletes the previous message, leaving a "Message deleted" bubble in its place.

Command Description
$delete Deletes the previous message on the next click
$delete = 3000 Auto-deletes the previous message after 3000ms (no click needed)

Basic Example

gf : I hate you!
$delete
gf : Sorry, I didn't mean that...

Timed Deletion Example

gf : Oops, wrong person!
gf : $pics = secret.jpg
$delete = 2000
gf : Pretend you didn't see that üòÖ
How it works

With $delete, the user clicks to see the deletion happen. With $delete = TIME, the deletion happens automatically after the specified time, and the conversation continues without waiting.

Thinking

The $thinking command displays the MC's inner thoughts as an overlay on top of the chat. This is useful to show what the MC is thinking without sending an actual message.

$thinking
I don't know what's happening to her.
Is she trying to provoke something in me? $/
I need to try to dig deeper.

mc : Hey, are you okay?
Element Description
$thinking Starts the thinking overlay. All text below will be displayed as MC's thoughts.
$/ Separates thoughts into different bubbles. Each section before $/ appears in its own thought bubble.
End of thoughts When you write a normal message (like mc :), the thinking overlay closes and the conversation resumes.
Tip

Use $/ to break long thoughts into multiple bubbles. Each bubble appears one after the other, creating a more dynamic inner monologue.

Spy App Support

The $thinking command also works in the girlfriend's conversations within the Spy App. Since the MC scrolls freely through her messages (rather than advancing click by click), the behavior is adapted: the thought overlay appears automatically when the MC scrolls past the trigger point, and the scroll is temporarily locked until he dismisses the thought bubble.

Choices

Fake Choices (No Impact)

This function allows the MC to submit a response of their choice, but it has no impact on the story. It is intended to enhance immersion.

$fake.choices =
Choice 1. $/
Choice 2. $/
Choice 3.
Syntax

You can include as many choices as you wish, but it is important that each line ends with $/, except for the last one.

Custom Responses

You can add a personalized response from an interlocutor for each choice:

gf : What do you want to do tonight?
$fake.choices =
Watch a movie. $/ [gf : Great idea! I'll prepare the popcorn!]
Go out for dinner. $/ [gf : Perfect, I know a nice restaurant!]
Stay home and relax. $/ [gf : Sounds cozy, I like it.]
I don't know, you choose.
Note

Custom responses are completely optional. You don't need to add one to every choice‚Äîmix and match as needed.

Real Choices (With Impact)

Here is an example of a choice-based feature that impacts the story:

$choices =
A. Choice 1 $/
B. Choice 2 $/
C. Choice 3

path a
gf : We're in timeline 1!
$talks = path1/example.txt
end path

path b
gf : We're in timeline 2!
$talks = path2/example.txt
end path

path c
gf : We're in timeline 3!
$talks = path3/example.txt
end path
Element Description
A., B., 1., 2.... Prefix to identify the choice (will not appear visually when making the choice)
path a ... end path Opens the continuation for choice A, and ends the path

Between these tags, you can create the conversation as you wish. You can even use $talks = example.txt to redirect the user to a discussion that only concerns that choice.

Choice Hints

You can add hints to help players understand the consequences of their choices. Hints appear as small badges on the right side of choice buttons.

Syntax Description
{hint text} Adds a gray hint badge
{$color:hint text} Adds a colored hint badge

Available Colors

You can use the following colors: red, green, blue, yellow, orange, purple, pink, cyan, gray (or grey), white.

Example

gf : What should we do tonight?
$choices =
A. Go to the party {$green:Good} $/
B. Stay home and rest {$blue:Neutral} $/
C. Ignore her message {$red:Bad}
User Setting

Players can toggle hints visibility in the Settings menu under "Choice hints". When disabled, hints are hidden but choices still work normally.

Best Practices

Use hints sparingly to guide players without spoiling the story. Common patterns include:

  • {$green:Good} / {$red:Bad} for moral choices
  • {$blue:Safe} / {$orange:Risky} for risk assessment
  • {$purple:Romance} / {$cyan:Friendship} for relationship paths

Locks

You can lock certain content behind a subscription code:

$lock = example.txt

Content of the lock file:

[GOLD]
$talks = chapter2/example.txt

Only members with a gold code (or higher) will be able to access this chapter.

Available Tiers
[BRONZE], [SILVER], [GOLD], [DIAMOND], [PLATINUM]
