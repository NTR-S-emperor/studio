Wallet

The Wallet app allows you to track financial transactions between characters in your story. It displays a balance and a history of transactions (credits and debits), making it perfect for stories involving money, debts, or financial situations.

Unlocking the Wallet

The Wallet app is not visible by default. It must be unlocked through the story using a command in the MC's Messenger conversation.

Basic Unlock

To unlock the Wallet with a $0 starting balance:

$wallet.unlock

Unlock with Initial Balance

You can set an initial balance when unlocking the Wallet. This is useful if you want the MC to start with some money:

$wallet.unlock.500

This unlocks the Wallet with a starting balance of +$500.00.

You can also use negative amounts or decimals:

$wallet.unlock.1000.50
$wallet.unlock.-200
Initial Balance

The initial balance does not appear as a transaction in the history. The user will simply see the balance displayed at the top of the Wallet when they open it.

Tip

When the Wallet is unlocked, a notification appears on the phone to inform the user that the app has been installed.

Adding Transactions

Transactions are added through Messenger commands. Each transaction can be a credit (positive amount) or a debit (negative amount).

Syntax

$wallet.AUTHOR = AMOUNT : LABEL : DATE
Parameter Required Description
AUTHOR Yes mc or gf - who initiated the transaction
AMOUNT Yes The amount (positive for credit, negative for debit)
LABEL Yes Description of the transaction
DATE No Optional date or time indication

Examples

Debit (spending money):

$wallet.mc = -100 : Bought a new graphics card : Yesterday

Credit (receiving money):

$wallet.gf = +50 : Pizza reimbursement

Transaction with decimals:

$wallet.gf = -29.99 : Restaurant bill : Dec 25

Display

Transactions are displayed in the Wallet with:

  • Green for credits (positive amounts)
  • Red for debits (negative amounts)
  • The author's name (using the customized character names if set)
  • The optional date
Author Names

The author field (mc or gf) determines whose name appears on the transaction. If the user has customized character names, those names will be displayed instead of the defaults.

Notifications

The Wallet app sends notifications for important events:

App Installation

When the Wallet is unlocked via $wallet.unlock, a notification appears to inform the user that the app has been installed.

New Transactions

Each time a transaction is added, a notification appears showing the amount and label:

  • Example: +$50.00 - Pizza reimbursement
  • Example: -$100.00 - Bought a new graphics card

Clicking on a notification opens the Wallet app directly.

Complete Example

Here's a complete example showing how to use the Wallet in a story:

Sarah
gf : Hey $mc! I need to ask you something...
mc : Sure, what's up?
gf : Can you lend me $200 for groceries? I'll pay you back!
mc : Of course, no problem.

$wallet.unlock.500
$wallet.mc = -200 : Loan to $gf : Today

gf : Thanks! You're the best!
mc : No worries.

[Later...]

gf : Here's $50 back from the groceries!
$wallet.gf = +50 : Partial reimbursement

mc : Thanks! Still waiting on the rest though...
gf : I know, I know... soon!
Storytelling Tip

The Wallet is great for tracking debts, shared expenses, or financial manipulation in your story. You can use it to create tension around money issues between characters.

Quick Reference

$wallet.unlock.AMOUNT Unlock with initial balance $wallet.unlock.500

$wallet.gf = AMOUNT : LABEL Add transaction from GF $wallet.gf = +50 : Refund
